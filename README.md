# MusesAOS

**面向 AI 原生内容生产的多模态 Agent 创作平台**

MusesAOS 将文本、图像、视频、语音、音乐生成和 AI Agent 执行能力整合到一个可视化工作区。它不是单点式的 AI 工具集合，而是一个可以把创意、模型、流程、资产和执行环境组织起来的生产系统。

![MusesAOS 首页截图](public/screenshots/musesaos-home.png)

## 一句话介绍

MusesAOS 帮助创作者、内容团队和 AI 应用团队，用可视化流程画布搭建可复用的多模态内容生产工作流，并通过 Agent 沙箱把“生成内容”推进到“执行任务”和“交付结果”。

## 为什么是现在

生成式 AI 已经从单次 prompt 进入长程工作流阶段。内容团队不再只需要一个聊天框或一个图片生成器，而是需要一个能够持续组织生产过程的平台：

- 多模型并存，需要统一入口和任务编排。
- 内容生产从单素材扩展到脚本、分镜、图片、视频、配音、音乐和发布素材的组合链路。
- AI Agent 开始承担执行型任务，需要安全、可观察、可回收的沙箱环境。
- 团队需要项目化管理生成历史、资产和流程，而不是把结果散落在多个工具中。

MusesAOS 的机会在于成为“AI 内容生产操作系统”：把模型能力产品化，把创意流程资产化，把 Agent 执行能力纳入可控工作区。

## 产品定位

MusesAOS 当前聚焦三个核心场景：

| 场景 | 用户价值 |
| --- | --- |
| 多模态内容生产 | 在同一画布中完成文本、图像、视频、语音和音乐生成，减少工具切换 |
| 可复用创作流程 | 将输入、模型、生成节点和结果串成工作流，让创作方法可以沉淀和复用 |
| Agent 执行与交付 | 通过沙箱让 Agent 执行命令、生成文件、下载产物，形成更完整的交付链路 |

## 核心能力

### 1. 可视化 AI 工作流画布

用户可以在画布上组合输入节点、生成节点和多模态节点，把一次性 prompt 变成可保存、可迭代、可复用的生产流程。

### 2. 多模态模型统一接入

平台已接入文本、图像、视频、语音和音乐模型，并通过统一的服务端 API 管理鉴权、上传、错误处理和积分计费。

当前模型目录包括：

- 文本：GPT-5 Mini、DeepSeek V4 Flash、DeepSeek V4 Pro、Kimi K2.5、Doubao Seed 1.8
- 图像：Doubao Seedream 5.0 Lite、Gemini 3 Pro Image、Gemini 2.5 Flash Image、GPT Image 2
- 视频：Kling、Doubao、Seedance 2.0、HappyHorse 1.0
- 语音：Speech 2.6 HD
- 音乐：Suno V5

### 3. AI Agent 与沙箱执行

MusesAOS 内置 Agent 面板，支持流式对话、工具执行事件追踪、运行时复用和上下文重置。Agent 可以启动 E2B Node.js 22+ 沙箱，执行命令、读写文件，并将产物回传到用户工作区。

### 4. 项目与资产管理

平台支持项目创建、保存、加载、重命名和画布状态持久化。生成资产通过服务端上传到数据库，前端仅接收可访问的资源 URL，便于后续沉淀为团队资产库。

### 5. 计费与风控基础

生成任务已经具备积分预扣、确认和失败回滚能力。关键接口具备输入校验、鉴权、限流和日志脱敏，为后续商业化和规模化使用打基础。

## 差异化优势

### 工作流优先，而不是工具堆叠

MusesAOS 的核心不是“接入更多模型”，而是把模型组织成流程。用户可以把创作链路沉淀为项目和资产，而不是每次从零开始。

### 多模态与 Agent 融合

多数创作工具停留在内容生成层。MusesAOS 将 Agent 执行、沙箱文件、工具事件和生成节点放在同一系统中，让平台具备从“生成素材”走向“完成任务”的能力。


### 可扩展模型架构

模型目录集中配置，生成能力通过服务端路由封装。新增模型、替换供应商或引入企业私有模型，不需要重写前端工作流。

## 本地运行

```bash
npm install
cp .env.example .env.local
npm run dev
```

默认访问 [http://localhost:3000](http://localhost:3000)。如果端口被占用，Next.js 会自动切换到可用端口。

关键环境变量包括 Supabase、DeepSeek、Muses Agent 和 E2B 配置。详见 `.env.example`。
