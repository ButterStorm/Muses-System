## Design Context

### Users
Muses System 是一个 AI 驱动的多模态创作平台，用户是创意工作者、内容创作者，他们使用流程图式交互来生成文本、图片、视频、音频等内容。他们在画布上专注创作，需要 AI 助手随时待命但不打扰。

### Brand Personality
**Voice**: 专业但亲切，现代但不冰冷
**Tone**: 优雅、专注、智能
**3-word personality**: 柔和、优雅、沉浸
**Emotional goals**: 让用户感到被支持而不是被打扰，AI 助手像一个安静的创意伙伴

### Aesthetic Direction
**Visual tone**: 柔和优雅 - 毛玻璃质感、柔和阴影的现代风格
**Theme**: 与画布灰色背景融合的浅色毛玻璃面板，而非突兀的亮色
**References**: macOS 系统级 UI、Linear 的精致感、Notion 的沉浸体验
**Anti-references**: 鲜艳渐变色、高对比度亮色面板、过多的装饰元素

### Design Principles
1. **融入而非突兀** - Agent 面板应该是画布的自然延伸，而非一个"贴上去"的组件
2. **柔和的视觉层次** - 使用毛玻璃、轻微阴影、微妙的边框来创造层次，而非强烈的颜色对比
3. **专注优先** - 最小化视觉干扰，让用户专注于创作流程
4. **响应即反馈** - 交互反馈要即时且优雅，但不喧宾夺主
5. **克制的美学** - 少即是多，每个视觉元素都应有明确目的

### Color Palette
- **Surface**: 半透明白色/灰色毛玻璃 (rgba(255,255,255,0.7) / rgba(248,250,252,0.8))
- **Border**: 极浅灰色 (rgba(0,0,0,0.05))
- **Text Primary**: 深灰/炭黑 (#1f2937)
- **Text Secondary**: 中灰 (#6b7280)
- **Accent**: 柔和的靛蓝/紫色调，呼应品牌但更加柔和 (#6366f1)
- **User Message**: 微妙的灰色调，而非鲜艳的蓝色
- **AI Message**: 纯白/极浅灰背景

### Typography
- **Font Family**: 系统字体栈，保持与项目一致 (-apple-system, BlinkMacSystemFont)
- **Hierarchy**: 清晰的层级，但不使用过大或过重的字体

### Spacing & Layout
- **Panel Width**: 保持 380px，但使用更柔和的过渡
- **Border Radius**: 统一使用圆角 (12-16px)
- **Padding**: 舒适的内边距，让内容呼吸
- **Shadow**: 柔和的多层阴影，创造深度感而非突兀感
